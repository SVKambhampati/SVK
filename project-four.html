StyleForecast

A full-stack outfit recommendation engine that fuses live weather telemetry with a user-managed wardrobe to surface ranked outfit suggestions for any conditions.

Flask Python SQLite OpenWeatherMap API Full-Stack
Visit Live Site ↗
thestyleforecast.vercel.app

Overview

StyleForecast is a full-stack web application that takes the guesswork out of getting dressed. It pulls real-time weather data for the user's location, cross-references it against their saved wardrobe, and surfaces outfit suggestions that are appropriate for the day's conditions and aligned with their personal style preferences.

How It Works

Users build a digital wardrobe by uploading clothing items with metadata — type, color, formality level, and season suitability. StyleForecast fetches the current forecast from the OpenWeatherMap API, runs the wardrobe through a recommendation engine that weights weather appropriateness and style preferences against temperature, precipitation, and UV index, and presents a ranked list of outfit options. Users can save favorites and rate suggestions to improve future recommendations.

Technologies Used

  • Python & Flask — backend logic and REST API server
  • SQLite — lightweight database for user profiles, wardrobe items, and weather logs
  • OpenWeatherMap API — real-time weather data by location
  • Jinja2 — server-side templating for the web interface
  • HTML, CSS & JavaScript — responsive frontend with dynamic outfit updates

What I Learned

StyleForecast was my first end-to-end full-stack project — from database schema design to API integration to frontend rendering. The biggest challenge was building a recommendation engine that felt genuinely useful rather than just random. It pushed me to think carefully about how to model user preferences and how small UX decisions dramatically change how trustworthy a recommendation feels.
