AddressSync Automation

A backend integration tool that ensures accurate shipping data between Oracle EBS and Oracle Commerce Cloud — automatically detecting and fixing address mismatches before they cause failed deliveries.

Oracle EBS Oracle Commerce Cloud PL/SQL API Integration Backend

The Problem

DeKalb County's IT systems were experiencing a recurring issue: shipping addresses entered in Oracle Commerce Cloud weren't always syncing correctly with Oracle EBS, the county's core enterprise system. The mismatch caused failed FedEx deliveries, manual correction work, and operational delays. There was no automated way to catch these errors before shipment.

The Solution

I built a backend integration layer that automatically compares address records between Oracle EBS and Oracle Commerce Cloud at the point of order processing. When a mismatch is detected, the system flags the order and attempts reconciliation before it reaches the fulfillment pipeline — stopping the problem at the source rather than dealing with the fallout after failed delivery.

Technologies Used

  • Oracle EBS — enterprise backend system holding canonical address records
  • Oracle Commerce Cloud — customer-facing storefront where orders originate
  • PL/SQL — validation scripts for address cleaning and standardization
  • REST API integration — real-time sync between both Oracle systems
  • FedEx shipping API — downstream validation for delivery address format

Impact

The integration significantly reduced failed delivery rates and eliminated the manual correction process that the logistics team had been managing. It also surfaced a broader data quality issue in how addresses were being entered on the storefront side, leading to upstream improvements in form validation.
