SentimentSignal

A real-time market sentiment engine that ingests social media streams, classifies posts as bullish, bearish, or neutral using an NLP pipeline, and aggregates per-ticker scores across 50+ equities.

Python NLP REST APIs Real-time Data Pipelines
GitHub ↗

Overview

SentimentSignal is a real-time market intelligence tool built at Oasis@Northeastern. It ingests a continuous stream of social media posts from Reddit, Twitter/X, and StockTwits, runs each post through an NLP classification pipeline to label it bullish, bearish, or neutral, and aggregates the results into per-ticker sentiment scores across 50+ equities. The goal is to surface momentum shifts in market sentiment before they appear in price action.

How It Works

Posts are pulled via API streams and queued for processing. Each post is tokenized, cleaned, and passed through a fine-tuned sentiment classifier trained on financial social media data. The classifier output is weighted by source credibility and post engagement, then rolled into a sliding-window aggregation per ticker symbol. A live dashboard surfaces the current sentiment distribution and momentum trend for each tracked equity.

Technical Decisions

  • Multi-source ingestion: Reddit (PRAW), Twitter/X API v2, and StockTwits feed API — normalized into a common schema before hitting the pipeline.
  • NLP classifier: Fine-tuned on FinSentiment and StockTwits labeled datasets. Outperforms vanilla VADER on finance-specific language like options slang and ticker mentions.
  • Sliding window aggregation: Configurable time windows (15m, 1h, 4h) let users distinguish short-term noise from sustained sentiment shifts.
  • Low-latency design: Pipeline target under 500ms from ingest to dashboard update under normal load.

Technologies Used

  • Python — core pipeline and classifier logic
  • Hugging Face Transformers — fine-tuned sentiment classification model
  • PRAW / Twitter API v2 / StockTwits API — multi-source social data ingestion
  • SQLite / JSON store — ticker-level sentiment aggregation and history
  • Flask — lightweight dashboard API server

What I Learned

The biggest challenge was financial language. General-purpose sentiment models completely miss context like "puts printing" or "to the moon" — which meant investing in a domain-specific training set. I also learned a lot about pipeline reliability: API rate limits, partial failures, and deduplication across sources all needed explicit handling to keep the signal clean. Building this made clear how much infrastructure work lives beneath a "simple" real-time dashboard.
